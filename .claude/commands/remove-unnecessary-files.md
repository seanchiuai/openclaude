# Remove Unnecessary Files

1. Read all files in `/docs/` (including `CHANGELOG.md`).
2. For each file, evaluate whether its content is useful for future development:
   - **Keep**: Architecture decisions, API contracts, environment setup, deployment procedures
   - **Remove**: Outdated changelogs, resolved incident notes, historical meeting notes, stale planning docs
3. For files to clean up, remove the unnecessary sections/entries rather than deleting the entire file.
4. Present a summary of what you plan to remove and ask for confirmation before making changes.
